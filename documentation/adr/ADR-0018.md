## Title

Add Netdiscover as a Network Reconnaissance Tool

## Proposal status

What is the status of this decision?

-   Proposed

## Implementation status

-   Work in progress

## Context

Our current penetration testing toolkit lacks a lightweight, passive network discovery utility suitable for identifying live hosts in a subnet without generating high traffic. Tools like Nmap offer active scanning but may be too intrusive for stealth operations or controlled environments. During internal testing, we identified several scenarios where passive discovery was necessary, especially in isolated network segments with minimal logging to avoid detection. The goal is to enhance our reconnaissance capabilities with a tool that is quick, minimal, and effective in low-noise environments.

## Decision

We will integrate Netdiscover into our existing toolkit as the go-to passive ARP-based scanner. Netdiscover is effective for identifying live hosts on a local network using ARP requests. It is particularly useful in networks without DHCP servers or where DNS resolution is unavailable. We will create documentation, usage instructions, and safety guidelines for using Netdiscover and incorporate it into our tool launcher interface alongside other reconnaissance tools.

## Consequences

Positive:

Enables stealthy host discovery without alerting IDS/IPS systems.

Lightweight and easy to run on limited-resource environments.

Adds value to the toolkit for red team and network auditing tasks.

Supports use cases in isolated network segments or closed environments.


Potential Drawbacks:

Only works effectively on local subnets (Layer 2); not useful across routed networks.

May produce incomplete results on networks with static ARP tables or MAC filtering.

Requires root privileges which may pose restrictions in some environments.
