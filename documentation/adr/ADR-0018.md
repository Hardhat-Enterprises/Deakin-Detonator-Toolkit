## Title

Add Netdiscover as a Network Reconnaissance Tool

## Proposal status

What is the status of this decision?

-   Approved

## Implementation status

-   Work in progress

## Context

The current penetration testing toolkit lacks a lightweight, passive network discovery utility suitable for identifying live hosts in a subnet without generating high traffic. Tools like Nmap offer active scanning but may be too intrusive for stealth operations or controlled environments. Internal testing revealed multiple scenarios where passive discovery was necessary, particularly in isolated network segments with minimal logging to avoid detection. The aim is to enhance reconnaissance capabilities with a tool that is quick, minimal, and effective in low-noise environments.

## Decision

Netdiscover will be integrated into the existing toolkit as the default passive ARP-based scanner. This tool is effective in identifying live hosts on a local network using ARP requests and is especially useful in networks lacking DHCP servers or DNS resolution. Documentation, usage instructions, and safety guidelines will be developed to support Netdiscover and it will be included in the tool launcher interface alongside other reconnaissance utilities.

## Consequences

Positive:

Enables stealthy host discovery without alerting IDS/IPS systems.

Lightweight and easy to run in limited-resource environments.

Enhances the toolkit for red team and network auditing tasks.

Supports use in isolated network segments or closed environments.

Potential Drawbacks:

Only effective on local subnets (Layer 2); not suitable across routed networks.

May yield incomplete results on networks with static ARP tables or MAC filtering.
